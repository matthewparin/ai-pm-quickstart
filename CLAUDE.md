This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## About this repo

This is an AI-assisted Product Management portfolio — a file/document-based workspace, not a software application. It houses PM artifacts demonstrating systematic AI tool deployment for product work.

**Directory layout:**
- `_context/` — Claude context and prompt engineering files
- `research/` — Product research, market analysis, user insights
- `workflows/` — PM processes and decision frameworks
- `prototypes/` — Product prototypes or prototype documentation
- `templates/` — Reusable templates (PRDs, briefs, specs)

## About the user

Senior Product Director at Salesforce leading Senior PMs and Directors. Domain focus: ISV/OEM partner products — telemetry, monetization, security/governance, extensibility, product growth, new product introduction. Goals: prototype products, evaluate team output, understand engineering decisions. Bias toward shipping small, iterative changes quickly.

## How to work

- Structured, direct output — no fluff
- Flag uncertainty rather than guessing
- Use Plan Mode before making any changes to existing files
- When reviewing PM work, assess: clarity of problem statement, quality of success metrics, feasibility of approach
- When writing PRDs, follow this structure: Problem, Users, Goals, Non-goals, Requirements, Open Questions

## Quality standards

- Every feature needs a clear success metric before scoping
- PRDs must reference user evidence, not assumptions
- Balance speed > quality > scale (in that priority order)
