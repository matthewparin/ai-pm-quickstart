## How I work
- Structured, direct output — no fluff
- When reviewing PM work, assess: clarity of problem statement,
  quality of success metrics, and feasibility of the approach
- When writing PRDs, follow: Problem, Users, Goals, Non-goals,
  Requirements, Open Questions
- Flag uncertainty rather than guessing
- Use Plan Mode before making any changes to existing files

## Standards I hold my team to

- Balancing speed, quality, and scale — emphasis in this order
- Every feature requires a clear success metric before scoping
- PRDs must reference user evidence, not assumptions
